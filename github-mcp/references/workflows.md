# GitHub MCP Workflows

## Issue Triage Workflow

1. Query open issues by label/assignee/state.
2. Group by severity, age, and owner gaps.
3. Propose label normalization and ownership assignments.
4. Apply metadata updates only to explicitly scoped issues.

## Pull Request Workflow

1. Read PR summary, checks, reviewers, and linked issues.
2. Identify blocking checks and review bottlenecks.
3. Add comments/reviewer requests with explicit rationale.
4. Prepare merge recommendation with risk and test status.

## Release Readiness Workflow

1. Collect PRs/issues for milestone or release label.
2. Highlight unresolved blockers and high-risk areas.
3. Produce go/no-go recommendation with concrete next steps.
